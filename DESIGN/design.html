Design Documentation

1. Project Overview

This website serves as a personal and professional digital portfolio for my university coursework and creative practice. It includes a homepage, blog, essays, a portfolio, and a design reflection section. The goal was to create a user-friendly, accessible, and visually appealing site that aligns with modern web standards.

2. User Research & Personas

The intended users are peers, lecturers, and potential creative collaborators. To cater to these audiences, I developed two simple personas:

  • Bella, 21: A Digital Arts student who visits the site on mobile to check blog content and artwork.
  • Dr. Smith, 42: A lecturer interested in evaluating how well theory is integrated into the site.

3. Design Goals

  • Use semantic HTML and accessible CSS for structure and clarity.
  • Ensure the site is responsive across screen sizes.
  • Highlight portfolio work in a visually engaging way.
  • Maintain a consistent visual identity through color and typography.

4. Information Architecture

I used a flat hierarchy, with major sections linked via the navbar: Home, Design, Blog, Portfolio, Essay, and Image page. This structure keeps navigation simple and content accessible with minimal clicks.

5. Wireframes & Layout

Initial wireframes were sketched using pen and paper, then digitized using Canva. Each page follows a grid system to ensure balance. I prioritized whitespace and alignment to guide the viewerâ€™s eye through each page.

6. UI Design Decisions

  • Typography: I used modern sans-serif fonts for readability and a clean aesthetic.
  • Color Scheme: A muted primary color paired with darker tones for contrast, inspired by minimalist portfolios.
  • Buttons and Links: Styled with hover effects for interactivity, using consistent padding and borders.

7. UX Considerations

  • Clear navigation via a sticky navbar.
  • Content is structured with headings and sections for scanability.
  • Portfolio images are large and clickable, enabling deeper engagement.
  • Blog and essays are readable with good contrast and line height.

8. Accessibility

I followed basic accessibility best practices, such as using semantic elements, descriptive link text, and high-contrast color palettes. Alt text is included for images, and font sizes are legible even on small screens.

9. Reflection & Iteration

Initially, I focused too much on aesthetics and overlooked metadata and folder organization. After receiving feedback, I restructured my files and ensured each page followed HTML5 standards. I also improved responsiveness by testing on mobile and adjusting layout accordingly.
