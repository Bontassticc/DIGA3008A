Accessibility Considerations

When I mentioned that my website is optimized for accessibility, I was referring to three core aspects of inclusive design: readability, contrast, and mobile responsiveness. Below, I break down how each of these is being addressed through specific design decisions.

Readability

  • Selected a Serif font ("Georgia, serif") for its clean, modern aesthetic and high legibility across screen sizes.
  • Body text size is set at 1rem with a line height of 1.6, promoting readability.
  • Used a consistent heading structure (H1 ‚Üí H2 ‚Üí H3) to support scanning and assistive technology navigation.
  • Avoided text justification to maintain even word spacing and support neurodiverse readers.

Color Contrast

  • Main text (#2c3e50) on light background (#f7e7ce) has a contrast ratio exceeding 12:1, meeting WCAG standards.
  • Accent color (#d8a7b1) provides emphasis while remaining accessible.
  • Hover states include visual indicators beyond color, such as underlines or slight shifts, to support colorblind users.

Mobile Responsiveness

  • Single-column layout adapts seamlessly to smaller screen sizes.
  • Images scale responsively using max-width: 100% and height: auto.
  • Spacing and padding are set in relative units (rem/em) for adaptability.
  • Navigation avoids hover-only functionality and features large tap targets suitable for touchscreens.

Interface Elements Updated

Buttons

Buttons on the site are used to prompt user interaction, such as navigating between pages, opening blog posts, and submitting forms. These buttons are consistently styled with rounded edges, a defined color palette, and hover animations to give users visual feedback. They often appear as call-to-action (CTA) elements, with clear labels like ‚ÄúRead More,‚Äù ‚ÄúView Project,‚Äù or ‚ÄúBack to Top.‚Äù The hover state slightly changes the background color and cursor to indicate interactivity, supporting accessibility and enhancing user control. On mobile, button sizes increase slightly for easier tapping, ensuring a seamless experience across all devices.

Cards

Card elements are used throughout the blog, portfolio, and homepage to group content into manageable, visually distinct sections. Each card includes a heading, an image or icon, a brief summary, and an action (like a link or button). Cards help organize information visually while maintaining a clean layout, enabling users to scan content quickly. Their use of shadows and spacing helps establish visual hierarchy and separation from surrounding elements. Cards scale responsively, stacking vertically on smaller screens to preserve readability and interactivity.

Navigation Bar

The navigation bar is a persistent UI component located at the top of the site. It includes links to core sections‚ÄîHomepage, Blog, Design, Essay, Profile, and Portfolio. On large screens, it is displayed horizontally, while on mobile, it collapses into a hamburger menu to conserve space. This responsive behavior ensures ease of navigation regardless of device. The active link is visually differentiated (e.g., underlined or colored) to indicate the current page, helping users maintain orientation within the site. The navigation bar is fixed in place, allowing constant access to site-wide movement.

Media Containers (Images, videos, Lightbox)

Images, wireframes, and potential videos are embedded inside stylized media containers that maintain alignment and proportion across devices. These containers may include borders, shadows, or padding to visually separate the media from surrounding text. Some images‚Äîparticularly in the design section‚Äîuse a lightbox effect that allows users to click and enlarge them without leaving the page. This enhances usability and allows for a deeper engagement with visual content, especially where annotations or small details are important.

10. Before & After: Iterative Design Changes

Homepage Layout

Before: The homepage used a large image header that pushed important content too far down. It lacked hierarchy, making the user's first impression unclear.

After: I reduced the image size, added a welcoming headline with a short intro, and included clear navigation buttons. This makes the purpose of the page more immediately visible and improves user flow.

Navigation Bar

Before: The navbar was not responsive and would overlap content on smaller screens. It also lacked visual clarity.

After: I implemented a responsive navbar with CSS media queries and added spacing and hover styles to enhance clarity and usability on both desktop and mobile devices.

Typography

Before: I used multiple font types and sizes inconsistently, which made the site feel unstructured and harder to read.

After:I limited my typefaces to one header font and one body font, standardized font sizes, and used consistent line spacing. This created a more cohesive and professional visual style.

Design Page Content

Before: The design page had only basic descriptions of the process with no clear structure or reflection on the design journey.

After: I expanded the section to include a detailed project overview, goals, personas, accessibility notes, and UX decisions. This better demonstrates the thinking behind my design choices and shows awareness of theory and practice.

Portfolio Display

Before: Portfolio items were listed in plain text, which made them visually unappealing and hard to browse.

After: I used image cards with hover effects and titles, creating a more interactive and engaging presentation that invites users to click and view each work in more detail.

File and Folder Organization

Before: My file structure was unorganized, with scattered assets and unclear file names, which made site management difficult.

After: I organized the website into clean folders for CSS, images, and JS, and renamed files for consistency. This not only improved workflow but also reflected professional web development practice.

Accessibility

Before: Alt text was missing from images and some pages lacked clear heading hierarchy.

After: I added descriptive alt text, semantic HTML elements like <main> and <section>, and made sure headings followed a logical structure (h1 ‚Üí h2 ‚Üí h3). This improved the accessibility and usability of the site.

New Additions and Improvements New

As the project progressed, I implemented several new features and enhancements aimed at improving both functionality and user experience across the website.

Modular JavaScript Enhancements: I refactored the JavaScript code to be more modular and maintainable. This involved breaking large scripts into smaller, reusable functions and centralizing event listeners to optimize performance and reduce redundancy. This approach also makes future updates easier and helps keep the codebase organized.

Interactive Lightbox Gallery: To showcase portfolio images more effectively, I added an interactive lightbox modal. This feature allows users to click on images to view larger versions in a modal window, navigate through images using arrow keys, and close the modal by clicking outside the image area or pressing the escape key, making the experience smooth and intuitive.

Reading Progress Indicator: On blog and essay pages, I implemented a scroll-based progress bar that visually tracks how much of the content the user has read. This small UX improvement helps orient readers and encourages engagement by giving clear feedback on reading progress.
