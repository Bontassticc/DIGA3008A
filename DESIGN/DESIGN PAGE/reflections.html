User Alignment

The kind of users I want to attract to my website are people interested in the work I do, such as animation (in the portfolio), developers and designers interested in scripts, and readers who want to engage with my blog content. Ideally, my audience includes potential collaborators or employers looking for someone with my skill set. To meet their needs, the site should be easy to navigate, visually appealing (aiming for a cute but elegant aesthetic), and well-structured.

Mapping Content, Experience & Interface

Content Mapping

  • Homepage: Introduction, site overview, "About Me" section, and a visual wall featuring my work.
  • Portfolio: Showcases my animation work and personal creative projects outside academics.
  • Blog: Reflections, updates, and additional written content.
  • Design: Showcases wireframes, color palettes, fonts, and UI concepts.
  • Scripts: Contains all JavaScript and CSS code used in the site.

Experience Mapping

  • Navigation: A consistent dashboard will be present on all pages for easy access.
  • User Journey: Lands on homepage → clicks "About Me" → scrolls to explore → visits portfolio → views work → contacts me.
  • Accessibility: Buttons include clear CTAs, intuitive layouts, and mobile-friendly design considerations.

Interface Elements

  • Navigation Bar: Links to main sections (Homepage, Blogs, Portfolio, Design, Scripts).
  • Buttons & CTAs: Clear actions like “Read More” and “View Portfolio”.

Structure, URL Scheme & Semantic Markup

URL Scheme

  • Portfolio: mywebsite.com/portfolio
  • Blog: mywebsite.com/blog
  • Individual Blog Posts: mywebsite.com/blog/blogposts
  • Design Page: mywebsite.com/design
  • Scripts Page: mywebsite.com/scripts

Semantic Markup

  • <header> – Used for navigation structure.
  • <section> – To group thematically related content.
  • <article> – Will be used for individual blog posts.

How It All Comes Together

  • User alignment ensures the site matches visitor expectations.
  • Content, experience, and interface mapping guides the user journey.
  • Semantic HTML and URL structure improve accessibility and organization.
