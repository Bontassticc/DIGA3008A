WEEK 3:BLOG POST

User Alignment

The kind of users that I want to attract to my website are people that are interested in the work that I do such as animation etc, which they will see in my portfolio; potential developers and designers looking for scripts, where they will see my script page; and readers that are interested in my blogs, reflection =s and the journey to creating this website. The goals that I want the viewer to have whilst being on my website, is the need and want to explore what I do and what I have done and ultimately be contacted for work or a service, so preferably the type of viewers I would want are those that are looking for someone who can provide a service. To align with the users my site will have to be easy to navigate, visually appealing (I’m trying to go for something cute but elegant) and clearly structured (no confusion about where to find things).

Mapping out Content, Experience and Interface Elements

Content Mapping

  • Hompage:Introduction, a section showing you all that you can explore, an “About Me” section and the wall drop being some of my own work
  • Portfolio Page:This is where I will showcase the work I’ve done in Animation and my own personal creations out of academics.
  • BlogPost Page:This is where I will have my blogposts that contain my reflections, updates and whatever else more I decide to add and or write about.
  • Design Page:In this section of the website is where i will showcase the ideas of the website and the updated wireframes, colour pallates, fonts and UI elements
  • Script Page: this is where all possible java scripts and maybe CSS scripts will be documented.

Experience Mapping

  • Navigation:On each page there will be a dashboard that will show the different pages that you can still access from there, no matter what page you're in.
  • User Journey: Landing on the homepage – clicks about me – scrolls down to explore page - clicks portfolio – explores work – contacts me
  • Accessibility and Useability:All buttons will have a call to action, in this way the whole page will be intuitive and we will eventually allow for it to be mobile friendly.

Interface Elements

  • Navigation Bar - links to Homepage, BlogPosts Page, Portfolio Page, Design Page, Script Page
  • Buttons and CTA's- "Read More", "View Portfolio"

Structre, URL Scheme, and Semantic Mark Up

Structure and URL Scheme

  • Portfolio Page:mywebsite.com/Portfolio
  • BlogPost Page:mywebsite.com/blog
  • Individual Blogs:mywebsite.com/blog/blogposts
  • Design Page:mywebsite.com/design
  • Script Page:mywebsite.com/scripts

Semantic Markup

  • Header - For Navigation
  • Section - Grouping Content
  • Article - Using in blogposts

How it all comes together

  • User alignment ensures your site meets visitors' needs.
  • Content, experience, and interface mapping creates a logical, engaging site structure.
  • Semantic markup and URL structure make your site more accessible and optimized.
